# Dart SDK API Documentation

Modern VitePress-based API documentation for the [Dart SDK](https://github.com/dart-lang/sdk).

Generated by [dartdoc-vitepress](https://github.com/777genius/dartdoc_vitepress) - a Dart documentation generator that produces VitePress sites.

## Live Site

**https://777genius.github.io/dart-sdk-api/**

## Features

- Full Dart SDK API reference with linked type signatures
- Full-text search across all libraries
- Dark/light mode
- Interactive DartPad embeds
- Collapsible outline for large API containers
- Copy button on all code blocks
- Responsive mobile layout

## Libraries

Core: `dart:core`, `dart:async`, `dart:collection`, `dart:convert`, `dart:math`, `dart:typed_data`, `dart:developer`

Web: `dart:html`, `dart:js_interop`, `dart:js_interop_unsafe`, `dart:ui`, `dart:ui_web`

## Quick Start

### Prerequisites

- [Dart SDK](https://dart.dev/get-dart) 3.10+
- [Node.js](https://nodejs.org/) 18+

### 1. Clone

```bash
git clone https://github.com/777genius/dart-sdk-api.git
cd dart-sdk-api
```

### 2. Generate API docs

Clone [dartdoc-vitepress](https://github.com/777genius/dartdoc_vitepress) and run:

```bash
git clone https://github.com/777genius/dartdoc_vitepress.git /tmp/dartdoc-vitepress
cd /tmp/dartdoc-vitepress && dart pub get

dart run /tmp/dartdoc-vitepress/bin/dartdoc_vitepress.dart \
  --sdk-docs --format vitepress --output /path/to/dart-sdk-api
```

### 3. Run dev server

```bash
npm install
npx vitepress dev --port 5174
```

Open http://localhost:5174/dart-sdk-api/

### 4. Build for production

```bash
NODE_OPTIONS="--max-old-space-size=24576" npx vitepress build
```

> **Note:** VitePress build requires ~16-24 GB RAM for 1800+ pages.

## CI/CD

Deployment is fully automated via GitHub Actions. Every push to `main` triggers:

1. Generate API docs from latest Dart SDK
2. Build VitePress static site (with 20GB swap for memory)
3. Deploy to GitHub Pages

See [`.github/workflows/deploy.yml`](.github/workflows/deploy.yml) for details.

To trigger a manual deploy: go to Actions > "Deploy to GitHub Pages" > Run workflow.
